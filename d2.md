### Python vs C — Side by Side

| Feature | C | Python |
|---------|---|--------|
| Compilation | `gcc file.c -o file` | Not needed, interpreted |
| Entry point | `int main()` | Just write code at the top |
| Print | `printf("Hello\n")` | `print("Hello")` |
| Variables | `int x = 5;` | `x = 5` |
| Blocks | `{ }` | Indentation (spaces) |
| Semicolons | Required `;` | Not needed |
| Strings | `char str[]` | `str = "hello"` |

> **Important:** Python uses **indentation** (4 spaces) instead of `{}` to define blocks. This is the #1 mistake beginners make coming from C.

## Before Exercise 1 — things that will break your code

1. input() ALWAYS returns a STRING — convert it!
   ```python
   age = input("Age: ")         # "20"  — this is text, not a number
   age = int(input("Age: "))    # 20    — now it is a number
   price = float(input("Price: "))  # 20.5 — use float for decimals
   ```

2. Python has no ++ operator
   ```python
   # Wrong:  count++
   # Right:
   count += 1
   ```

3. Indentation IS the code — always 4 spaces
   ```python
   # Wrong (IndentationError!):
   if x > 0:
   print("positive")   # not indented

   # Right:
   if x > 0:
       print("positive")  # 4 spaces indent
   ```

4. Division always returns float in Python 3
   ```python
   10 / 2   # ->  5.0  (float)
   10 // 2  # ->  5    (integer — use // for whole number division)
   ```

### Project A — Shopping Cart Simulator

Build this from scratch. The full reference implementation is below.



```python
# shopping_cart.py

class Product:
    def __init__(self, id, name, price, stock):
        self.id = id
        self.name = name
        self.price = price
        self.stock = stock

    def __str__(self):
        return f"{self.name} — ${self.price:.2f}"


class ShoppingCart:
    def __init__(self):
        self.items = {}   # {product_id: {"product": Product, "qty": int}}

    def add(self, product, qty=1):
        if product.stock < qty:
            print(f"Not enough stock for {product.name}")
            return
        if product.id in self.items:
            self.items[product.id]["qty"] += qty
        else:
            self.items[product.id] = {"product": product, "qty": qty}
        print(f"Added {qty}x {product.name} to cart")

    def remove(self, product_id):
        if product_id in self.items:
            del self.items[product_id]
            print("Item removed")
        else:
            print("Item not in cart")

    def total(self):
        return sum(
            item["product"].price * item["qty"]
            for item in self.items.values()
        )

    def show(self):
        if not self.items:
            print("Your cart is empty.")
            return
        print("\n--- Your Cart ---")
        for item in self.items.values():
            p = item["product"]
            q = item["qty"]
            print(f"  {p.name} x{q}  = ${p.price * q:.2f}")
        print(f"  TOTAL: ${self.total():.2f}")
        print("-----------------\n")


# Test it
catalog = [
    Product(1, "T-Shirt", 20.0, 50),
    Product(2, "Jeans",   50.0, 10),
    Product(3, "Shoes",   40.0, 0),
]

cart = ShoppingCart()
cart.add(catalog[0], 2)
cart.add(catalog[1], 1)
cart.add(catalog[2], 1)   # should warn — out of stock
cart.show()
```

**Expected output:**
```
Added 2x T-Shirt to cart
Added 1x Jeans to cart
Not enough stock for Shoes

--- Your Cart ---
  T-Shirt x2  = $40.00
  Jeans x1  = $50.00
  TOTAL: $90.00
-----------------
```

**What to look for:**
- `Product` class with `__init__` and `__str__`
- `ShoppingCart` uses a dict keyed by `product.id` (not a list) — enables O(1) lookup
- `add()` checks stock before adding; increments qty if item already in cart
- `total()` uses `sum()` with a generator expression
- `show()` handles the empty cart case with an early return



### Project B — Product Search CLI

Build this from scratch. The full reference implementation is below.


```python
# product_search.py

import json

products = [
    {"id": 1, "name": "T-Shirt",  "price": 20.0, "category": "clothing"},
    {"id": 2, "name": "Jeans",    "price": 50.0, "category": "clothing"},
    {"id": 3, "name": "Shoes",    "price": 40.0, "category": "footwear"},
    {"id": 4, "name": "Sneakers", "price": 60.0, "category": "footwear"},
    {"id": 5, "name": "Hat",      "price": 15.0, "category": "accessories"},
]

def search_by_name(query):
    query = query.lower()
    return [p for p in products if query in p["name"].lower()]

def filter_by_category(category):
    return [p for p in products if p["category"] == category]

def filter_by_price(max_price):
    return [p for p in products if p["price"] <= max_price]

def display(items):
    if not items:
        print("No products found.")
        return
    for p in items:
        print(f"  [{p['id']}] {p['name']} — ${p['price']} ({p['category']})")

while True:
    print("\n1. Search by name")
    print("2. Filter by category")
    print("3. Filter by max price")
    print("4. Show all")
    print("0. Exit")
    choice = input("Choose: ")

    if choice == "1":
        q = input("Search: ")
        display(search_by_name(q))
    elif choice == "2":
        cat = input("Category (clothing/footwear/accessories): ")
        display(filter_by_category(cat))
    elif choice == "3":
        max_p = float(input("Max price: $"))
        display(filter_by_price(max_p))
    elif choice == "4":
        display(products)
    elif choice == "0":
        print("Goodbye!")
        break
    else:
        print("Invalid choice")
```

**What to look for:**
- Each search/filter is its own function — not one massive `if` block
- `query.lower()` and `.lower()` on the product name so search is case-insensitive
- List comprehension `[p for p in products if ...]` used in all three filter functions
- `while True` loop with `break` on `"0"` to exit cleanly
- `display()` handles the empty result case

**1 & 2 — Save and load the cart with JSON**

Add these two functions to `shopping_cart.py` and call them at the right points:

```python
import json
import os

CART_FILE = "cart.json"

def save_cart(cart):
    """Save the cart items dict to a JSON file."""
    # We can't serialize Product objects directly, so save only the data we need
    serializable = {
        pid: {"id": data["product"].id,
              "name": data["product"].name,
              "price": data["product"].price,
              "qty": data["qty"]}
        for pid, data in cart.items.items()
    }
    with open(CART_FILE, "w") as f:
        json.dump(serializable, f, indent=2)
    print("Cart saved.")

def load_cart(cart, catalog):
    """Load previously saved cart items from JSON."""
    if not os.path.exists(CART_FILE):
        return   # nothing to load on first run
    with open(CART_FILE, "r") as f:
        saved = json.load(f)
    for pid_str, item in saved.items():
        # Find the matching Product object from the catalog
        product = next((p for p in catalog if p.id == item["id"]), None)
        if product:
            cart.items[product.id] = {"product": product, "qty": item["qty"]}
    print("Cart loaded from previous session.")
```

**3 — Apply a percentage discount to the entire cart**

Add this method inside the `ShoppingCart` class:

```python
def apply_discount(self, percent):
    """Reduce the price of every item in the cart by the given percentage."""
    if not 0 < percent < 100:
        print("Discount must be between 1 and 99 percent.")
        return
    for item in self.items.values():
        item["product"].price = round(item["product"].price * (1 - percent / 100), 2)
    print(f"{percent}% discount applied to all items.")
```

Test it:
