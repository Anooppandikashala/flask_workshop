## Session 1 — Flask Setup

### What is Flask?
Flask is a **web framework** for Python. It lets you write Python code that responds to browser requests.

> **Analogy:** Think of Flask as a traffic controller. The browser sends a request to a URL — Flask decides which Python function handles it and what to send back.

## Session 2 — Flask Fundamentals

### Hello World in Flask

Create `app.py`:
```python
from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello, World!"

if __name__ == "__main__":
    app.run(debug=True)
```

Run it:
```bash
python3 app.py
```
Open browser: `http://127.0.0.1:5000`

> `debug=True` — auto-reloads when you save. Never use in production.

### Routes — Mapping URLs to Functions

```python
@app.route("/")
def home():
    return "Home Page"

@app.route("/about")
def about():
    return "About Page"

@app.route("/products")
def products():
    return "Products Page"
```

> **The `@app.route()` decorator** tells Flask: "when the browser visits this URL, run this function."

### URL Parameters — Dynamic Routes

```python
# /product/1 → shows product with ID 1
# /product/42 → shows product with ID 42
@app.route("/product/<int:product_id>")
def product_detail(product_id):
    return f"Showing product ID: {product_id}"

# String parameter
@app.route("/category/<category_name>")
def category(category_name):
    return f"Category: {category_name}"
```

### HTTP Methods — GET and POST

```python
from flask import request

@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        # Form was submitted — process data
        username = request.form.get("username")
        password = request.form.get("password")
        return f"Logged in as: {username}"
    else:
        # Browser visiting the page — show the form
        return "Show login form here"
```

### Exercise 1 — Routes Practice
Add these routes to `app.py`:
1. `/` — returns "Welcome to QuickShop!"
2. `/products` — returns "All Products"
3. `/product/<int:id>` — returns "Product ID: {id}"
4. `/api/hello` — returns JSON `{"message": "Hello from Flask!"}`


```python
from flask import Flask, jsonify

app = Flask(__name__)

@app.route("/")
def home():
    return "Welcome to QuickShop!"

@app.route("/products")
def products():
    return "All Products"

@app.route("/product/<int:id>")
def product_detail(id):
    return f"Product ID: {id}"

@app.route("/api/hello")
def api_hello():
    return jsonify({"message": "Hello from Flask!"})

if __name__ == "__main__":
    app.run(debug=True)
```

**How to verify each route works:**

| Visit in browser | Expected response |
|-----------------|-------------------|
| `http://127.0.0.1:5000/` | `Welcome to QuickShop!` |
| `http://127.0.0.1:5000/products` | `All Products` |
| `http://127.0.0.1:5000/product/5` | `Product ID: 5` |
| `http://127.0.0.1:5000/api/hello` | `{"message": "Hello from Flask!"}` |

**What to look for:**
- `@app.route(...)` decorator above each function — not inside or after
- Function names are unique (cannot have two functions named the same)
- `<int:id>` in the URL pattern — the `int:` prefix makes Flask convert it to an integer automatically
- `jsonify()` is imported from flask and used (not `json.dumps`)
- `app.run(debug=True)` only runs when the script is executed directly

## Session 3 — Jinja2 Templates

### What is Jinja2?
Jinja2 is Flask's **templating engine**. It lets you write HTML with Python-like expressions inside {% raw %}`{{ }}`{% endraw %} and {% raw %}`{% %}`{% endraw %}.

> **Analogy:** Like C macros — but more powerful. The template is a blueprint; Jinja fills in the values.

### Rendering a Template

```python
from flask import render_template

@app.route("/")
def home():
    return render_template("index.html", title="QuickShop")
```

Flask looks for `index.html` inside the `templates/` folder automatically.

## Session 4 — Building a REST API

### What is a REST API?
An API (Application Programming Interface) lets programs talk to each other over HTTP.  
Flask can serve both HTML pages and JSON APIs from the same app.

| HTTP Method | Action | Example URL |
|-------------|--------|-------------|
| GET | Read data | `GET /api/products` |
| POST | Create data | `POST /api/products` |
| PUT | Update data | `PUT /api/products/1` |
| DELETE | Delete data | `DELETE /api/products/1` |

### Building the Products API

```python
# app.py — API section

products_db = [
    {"id": 1, "name": "T-Shirt",  "price": 20.0, "stock": 50, "category": "clothing"},
    {"id": 2, "name": "Jeans",    "price": 50.0, "stock": 30, "category": "clothing"},
    {"id": 3, "name": "Shoes",    "price": 40.0, "stock": 20, "category": "footwear"},
]

# GET all products
@app.route("/api/products", methods=["GET"])
def api_get_products():
    return jsonify(products_db)


# GET single product (using for loop)
@app.route("/api/products/<int:product_id>", methods=["GET"])
def api_get_product(product_id):
    product = None

    for p in products_db:
        if p["id"] == product_id:
            product = p
            break

    if product is None:
        return jsonify({"error": "Product not found"}), 404

    return jsonify(product)


# POST — create a new product (using for loop to find max id)
@app.route("/api/products", methods=["POST"])
def api_create_product():
    data = request.get_json()

    if not data or "name" not in data or "price" not in data:
        return jsonify({"error": "name and price are required"}), 400

    max_id = 0
    for p in products_db:
        if p["id"] > max_id:
            max_id = p["id"]

    new_product = {
        "id": max_id + 1,
        "name": data["name"],
        "price": data["price"],
        "stock": data.get("stock", 0),
        "category": data.get("category", "general"),
    }

    products_db.append(new_product)

    return jsonify(new_product), 201


# DELETE a product (using for loop)
@app.route("/api/products/<int:product_id>", methods=["DELETE"])
def api_delete_product(product_id):
    global products_db

    found = False
    new_list = []

    for p in products_db:
        if p["id"] == product_id:
            found = True
            continue
        new_list.append(p)

    if not found:
        return jsonify({"error": "Product not found"}), 404

    products_db = new_list

    return jsonify({"message": "Product deleted"})

```

@app.route("/products")
def products():
    all_products = load_products()
    category = request.args.get("category")   # /products?category=clothing

    # Filter products using for loop
    if category:
        filtered_products = []
        for p in all_products:
            if p["category"] == category:
                filtered_products.append(p)
        all_products = filtered_products

    # Get unique categories using for loop
    categories_set = set()
    all_products_data = load_products()

    for p in all_products_data:
        categories_set.add(p["category"])

    categories = []
    for c in categories_set:
        categories.append(c)

    return render_template(
        "products.html",
        products=all_products,
        categories=categories,
        selected_category=category
    )

## Session 6 — E-Commerce App Part 2: Cart

### Sessions in Flask
Flask's `session` object stores data **per user** in a browser cookie.  
We'll use it to store the cart without a database.

```python
from flask import session, flash

app.secret_key = "quickshop-secret-2025"   # Required for sessions
```

### Cart Routes

```python
@app.route("/cart/add/<int:product_id>", methods=["POST"])
def add_to_cart(product_id):
    all_products = load_products()

    # Find product using for loop
    product = None
    for p in all_products:
        if p["id"] == product_id:
            product = p
            break

    if product is None:
        flash("Product not found.", "danger")
        return redirect(url_for("products"))

    if product["stock"] == 0:
        flash("Sorry, this product is out of stock.", "warning")
        return redirect(url_for("products"))

    qty = int(request.form.get("qty", 1))

    # Initialize cart in session if it doesn't exist
    if "cart" not in session:
        session["cart"] = {}

    cart = session["cart"]
    product_id_str = str(product_id)  # session keys must be strings

    if product_id_str in cart:
        cart[product_id_str]["qty"] += qty
    else:
        cart[product_id_str] = {
            "id": product["id"],
            "name": product["name"],
            "price": product["price"],
            "qty": qty
        }

    session["cart"] = cart
    flash(f"'{product['name']}' added to cart!", "success")
    return redirect(url_for("products"))


@app.route("/cart")
def cart():
    cart = session.get("cart", {})

    # Calculate total using for loop
    total = 0
    for key in cart:
        item = cart[key]
        total += item["price"] * item["qty"]

    return render_template("cart.html", cart=cart, total=total)


@app.route("/cart/remove/<int:product_id>", methods=["POST"])
def remove_from_cart(product_id):
    cart = session.get("cart", {})
    product_id_str = str(product_id)

    if product_id_str in cart:
        removed_name = cart[product_id_str]["name"]
        del cart[product_id_str]
        session["cart"] = cart
        flash(f"'{removed_name}' removed from cart.", "info")

    return redirect(url_for("cart"))


@app.route("/cart/clear", methods=["POST"])
def clear_cart():
    session.pop("cart", None)
    flash("Cart cleared.", "info")
    return redirect(url_for("cart"))
```

import json

app = Flask(__name__)
app.secret_key = "quickshop-secret-2025"

# --- Data ---

def load_products():

# --- Page Routes ---

@app.route("/")
def home():
    all_products = load_products()

    featured = []
    count = 0
    for p in all_products:
        if count < 3:
            featured.append(p)
            count += 1

**3 — What does {% raw %}`{{ product | tojson }}`{% endraw %} do?**

It converts the Python dict into a JSON string and renders it directly in the HTML. This is useful when you need to pass data from Flask into JavaScript.

Example — add this to `product.html` temporarily to see it:
{% raw %}
```html
<pre>{{ product | tojson(indent=2) }}</pre>
```
{% endraw %}

You will see the raw JSON for the product dict printed on the page. The real use case is passing data to JavaScript:

{% raw %}
```html
{% block scripts %}
<script>
  // Flask passes the product dict directly into JS as an object
  const product = {{ product | tojson }};
  console.log(product.name);   // "Classic T-Shirt"
  console.log(product.price);  // 20.0
</script>
{% endblock %}
```
{% endraw %}

This avoids making a separate API call just to get the product data on a page that already has it.
